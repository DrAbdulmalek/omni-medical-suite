## دمج القواميس الطبية

- [x] **محول BGL داخلي:** استخدام `packages/medical/bgl_converter/` لقراءة ملفات Babylon (.bgl) واستخراج المداخل وتحويلها إلى صيغ متعددة (JSON, DIC, CSV, SQLite).
- [x] **قارئ StarDict:** تطوير `packages/medical/stardict_reader.py` لقراءة ملفات `.ifo`, `.idx`, `.dict`, `.syn` مع دعم الترميزات المتعددة وصيغتي 32-bit و 64-bit.
- [x] **معالج ملفات TMX:** تطوير `packages/medical/tmx_processor.py` لاستيراد ملفات TMX 1.1-1.4، كشف المصطلحات الطبية، ومراجعة المداخل يدوياً.
- [x] **خط أنابيب القواميس الموحد:** تطوير `packages/medical/dictionary_pipeline.py` لتنسيق استيراد القواميس من جميع المصادر (BGL, StarDict, TMX, JSON, CSV) في قاعدة بيانات موحدة.
- [x] **مدير القواميس:** تطوير `packages/medical/dictionary_manager.py` لإدارة القواميس المستوردة (بحث، تصحيحات OCR، مصطلحات محمية، تصدير) مع دعم قاعدة بيانات SQLite.
- [x] **REST API للقواميس:** إنشاء `services/api/app/api/dictionary_routes.py` بنقاط نهاية للاستيراد والبحث والتصحيح والتصدير.
- [x] **قاعدة بيانات موسعة:** تحديث `init.sql` بجداول جديدة (dictionaries, medical_terms, ocr_corrections, protected_terms, term_synonyms, dictionary_import_log) مع فهارس و views.
- [ ] **تحويل عبر PyGlossary:** تكامل اختياري مع PyGlossary لتحويل BGL إلى StarDict مباشرة (يستخدم في `DictionaryPipeline.convert_bgl_to_stardict()`).
- [ ] **واجهة مراجعة TMX تفاعلية:** صفحة ويب لمراجعة ملفات TMX يدوياً مع دعم اقتراحات الذكاء الاصطناعي للتصحيح.
- [ ] **التكامل مع NLP:** ربط القاموس بمراحل Correction و NER لتحسين تصحيح المصطلحات واستخراج الكيانات.
- [ ] **إثراء المستندات:** إضافة تعريفات المصطلحات الطبية تلقائياً كطبقة إثرائية عند معالجة النص.
- [ ] **دعم قواميس متخصصة:** إضافة قواميس تشريح، أدوية، أمراض نادرة كحزم جاهزة.

## تعزيز محرك التعرف البصري (OCR)

- [ ] **تحسين دعم الكتابة اليدوية العربية:** ضبط TrOCR بشكل أكبر على مجموعات بيانات خط اليد العربي الطبي (مثل READ أو KHATT).
- [ ] **استخراج الجداول المُحسَّن:** إضافة محرك Table Transformer من Microsoft إلى خط الأنابيب.
- [ ] **دعم ملفات PDF متعددة الصفحات بكفاءة أعلى:** معالجة متوازية لكل صفحة باستخدام Celery.
- [ ] **معالجة الصور المتدهورة:** دمج نموذج إزالة التشويش (Denoising Autoencoder) كمرحلة ما قبل المعالجة.
- [ ] **التعرف على الصيغ الكيميائية والرموز الطبية:** توسيع القدرات إلى رموز لا تتضمن نصوصاً فقط.

## تطوير معالجة اللغة الطبية (NLP)

- [ ] **نماذج NER طبية عربية متخصصة:** تدريب نموذج (مثل AraBERT v2) على بيانات طبية عربية موسومة لأسماء الأمراض والأدوية والفحوصات.
- [ ] **حماية سياق موسعة (Context Protector v2):** إضافة طبقة دلالية تستخدم معرفة طبية منظمة (مثل SNOMED CT) بدلاً من الاعتماد فقط على تشابه المتجهات.
- [ ] **التلخيص الطبي الذكي:** إنتاج ملخصات منظمة (SOAP Notes) بالعربية من النصوص السريرية الطويلة.
- [ ] **ترجمة طبية احترافية:** قناة ترجمة بين العربية والإنجليزية للمصطلحات الطبية باستخدام نماذج MarianMT مُضبوطة على الحقل الطبي.
- [ ] **كشف التضارب الدوائي:** تحليل الوصفات الطبية المستخرجة والإبلاغ عن التداخلات الخطيرة.

## تحسين الذكاء الاصطناعي وبوابة النماذج اللغوية

- [ ] **إضافة مزودين جدد:** دعم نماذج Gemini من Google، Claude 3 من Anthropic، و Command R+ من Cohere عبر بوابة LLM Gateway.
- [ ] **التوجيه الذكي (Smart Routing):** توجيه المهمة (ترجمة، تلخيص، استخراج) إلى النموذج الأفضل أداءً/تكلفةً وفقاً لسياق المستخدم.
- [ ] **التخزين المؤقت الذكي:** تخزين نتائج الاستعلامات المكررة لتقليل التكاليف (Semantic Cache).
- [ ] **التفاعل الصوتي:** إضافة إمكانية الإملاء الصوتي للنصوص الطبية العربية عبر Whisper API.

## تحسين تجربة المستخدم والواجهة

- [ ] **لوحة تحكم إحصائية (Dashboard):** رسوم بيانية لعدد المستندات المُعالجة والتصنيفات وأكثر المصطلحات شيوعاً.
- [ ] **التعاون في الوقت الحقيقي:** مشاركة رابط جلسة مع زميل لمراجعة تقرير مستخرج.
- [ ] **تحميل القوالب (Templates):** إنشاء قوالب تقارير طبية خاصة بالمؤسسة وتعبئتها تلقائياً.

## مقترحات عامة وتكاملات

- [ ] **واجهة برمجة تطبيقات عامة (Public API):** إصدار واجهة REST موثقة للسماح بتكامل السجلات الطبية الإلكترونية (EHR) مع النظام.
